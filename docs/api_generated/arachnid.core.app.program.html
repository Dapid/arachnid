program¶

Defines the program architecture for every application

This module defines a generic architecture shared among many scripts as well as providing shared utilities such as option parsing, option checking, option updating, MPI setup, multi-threading control, input filename updates, configuration file creation, logging setup, file processing and version control.

Essentially a script can utilize one of two program architectures:

  1. Batch processing
  2. Group or file processing

This is determined by whether the script defines a process() or batch() entry point.

By convention, a program should define a main entry point for execution from outside the script as follows:

def main():
    #Main entry point for this script
    from arachnid.core.app import program 
    program.run_hybrid_program(__name__, description = """Description of the script""")

if __name__ == "__main__": main()

Basically, it should define a main function that can serve as an entry point for setup.py. And, in case the script is executed from the command line, it should contain the following:

if __name__ == "__main__": main()

Usage¶

The target module must define one of the following functions:

process(filename, **extra)¶

Process a single file. During parallel processing this is invoked by a worker process.

See arachnid.core.app.file_processor for more information on the file or group processing architecture.

Parameters:
  • filename – Input filename to process
  • extra – Unused keyword arguments (Options from the command line or config file, plus additional options)
Returns:

At minimum filename, however a tuple of additional arguments can be returned for processing by reduce_all()
batch(files, **extra)¶

Process all input files.

Parameters:
  • files – List of input filenames to process
  • extra – Unused keyword arguments (Options from the command line or config file, plus additional options)

The target module must define all the the following:

setup_options(parser, pgroup=None, main_option=False)¶

Setup program options for command-line and configuration file

Parameters:
  • parser – OptionParser
  • pgroup – Root OptionGroup
  • main_option – If true, add the options specific to running this script

The target module may include any of the following functions:

change_option_defaults(parser)¶

Customize default options in other modules for this specific script.

Parameters:parser – OptionParser
dependents()¶

Return a list of dependent modules and add their options. Dependents of a dependent will also be recursively found and added.

Returns:List of dependent modules
update_options(options)¶

Utilize parsed options and add additional option values.

Parameters:options – object containing options as fields
check_options(options, main_option=False)¶

Test the validity of each option value and throw OptionValueError if an invalid value is found.

Parameters:
  • options – object containing options as fields
  • main_option – If true, add the options specific to running this script
flags()¶

Return a dictionary of support properties

Returns:Dictionary of supported properties

Parameters¶

The following parameters are added to a script using the program architecture.

--use-MPI <BOOL>¶

Set this flag True when using mpirun or mpiexec (Only available for MPI-enabled programs)

-z <FILENAME>, --create-cfg <FILENAME>¶

Create a configuration file (if the value is 1, true, y, t, then write to standard output)

--prog-version <version>¶

Version of the program to use, use latest to always use the latest version

--shared-scratch <FILENAME>¶

File directory accessible to all nodes to copy files (optional but recommended for MPI jobs)

--home-prefix <FILENAME>¶

File directory accessible to all nodes to copy files (optional but recommended for MPI jobs)

--local-scratch <FILENAME>¶

File directory on local node to copy files (optional but recommended for MPI jobs)

--local-temp <FILENAME>¶

File directory on local node for temporary files (optional but recommended for MPI jobs)

-t, --thread-count <INT>¶

Number of threads per machine, 0 means determine from environment (Default: 0)

run_hybrid_program
run_hybrid_program(name, **extra)¶

Main entry point for the program architecture

This function proceeds as follows:

  1. Determine module from name

  2. Determine if module supports file/group

    or batch processing

  3. Collect options, match to values and test validity using parse_and_check_options()

  4. Set number of threads for OpenMP (optional)

  5. Invoke entry point to file/group or batch processor

Parameters:
name : str

Name of calling module (usually __main__)

extra : dict

Pass on keyword arguments
