Development¶

See Arachnid API for code documentation.

Getting Source¶

To get the latest version of the source, use the following git command:

$ git clone https://code.google.com/p/arachnid/

Every time you update C or Cython code, you may run the following command (rather that reinstall)

$ python setup.py build_ext --inplace

Every time you add a new script or the first time you get the source, you may run the following command (rather that reinstall)

$ python setup.py develop --install-dir $BIN -m

Reporting Bugs¶

As a user or developer, you can report bugs or request features on the Google Issues Tab. Please, search for your issue before submitting it.

Contributing Code¶

The preferred way to contribute to Arachnid is to create your own local version or fork on code.google.com.

  1. Get a google account

  2. Create a google code project

  3. Make a local copy of this project

    $ git clone https://code.google.com/p/arachnid/

  4. Work on your copy locally using Git for version control

    # First time only, create a branch with label `my-feature`
$ git remote add origin https://my-username@code.google.com/p/arachnid-my-fork-name/
$ git push origin master
$ git checkout -b my-feature

# Run for each new file
$ git add modified_files

# Run for each major change
$ git commit

# Run to push to public respository (or your respository)
$ git push origin master

  5. When your feature is ready for release, create an issue

    • Use the Review Request template and include the URL to your project and a description and your contact.
    • You may be invited to act as a contributor or your code will be incorporated.

Your code should be error free and conform to the current code (also avoid import *). You can use the following tools to help ensure your code conforms to the proper standards.

  • Basic documentation and error checking

    $ easy_install pyflakes
$ pyflakes path/to/module.py

  • Unit test coverage

    $ easy_install nose coverage
$ nosetests --with-coverage path/to/tests_for_package

Debugging C/C++ code¶

Memory errors are the bain of every C/C++ developers existence. One important tool to finding these errors is valgrind.

  1. Install valgrind
  2. Download and modify the Python suppressions file: valgrind-python.supp See the directions in the README.valgrind for more information on modifying the suppressions.
  3. Run valgrind over your code (choose a small example because there is a performance cot)
$ valgrind -v --suppressions=valgrind-python.supp python my_test_script.py

Testing compilation of code¶

To test whether your code has any problems such as uninitalized variable, use the following flags in the environment for GCC type compilers.

$ export CFLAGS="-Werror -Wno-unused-function -Wno-unknown-pragmas -Wno-format" CXXFLAGS="-Werror -Wno-format -Wno-unknown-pragmas -Wno-unused-function"
$ export FFLAGS="-Werror -Wno-unused-function -Wtabs" F90FLAGS="-Werror -Wno-unused-function -Wtabs" F77FLAGS="-Werror -Wno-unused-function -Wtabs"

Packaging for Anaconda¶

# Clean out all non-package code
$ conda package -r

# Install non-package arachnid code
$ cd %arachnid$
$ python setup.py install

# Create distribution directory
$ mkdir dist
$ cd dist

# Create Package
$ conda package --pkg-name=arachnid --pkg-version=$version

# Create index (Optional, used for local respository)
$ conda index

Documentation Hack¶

To get the documentation to build correctly, you need to edit sphinx/ext/autosummary/generate.py in your site-packages directory.

Change Line 143 from

for name in dir(obj):

to

for name in vars(obj):

A little background: The default autosummary code gets all inherited members of a class. This ensures only the current members will be documented.

Custom Code¶

Arachnid supports two types of scripting: applications and fast prototyping. Applications are scripts that are designed for a user and to fit on the reconstruction workflow. Fast prototyping, however, is a way to test new ideas, perform a simple task quickly or customize objects such as plots.

The snippets cover a set of examples for writing fast-prototype code. These examples range from unstacking a SPIDER image stack to customizing the FSC plot.
